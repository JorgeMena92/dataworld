# DAX Patterns

Practical DAX patterns for the most common reporting requirements — aggregations, dynamic measures, field parameters, ranking, and conditional formatting values. These patterns are building blocks you will reuse across most Power BI projects.

---

## Basic Aggregations

The foundation of every model. Always create explicit measures rather than relying on implicit column aggregations — it gives you control over formatting, naming, and behavior.

```dax
Total Sales      = SUM(fact_sales[amount])
Total Orders     = COUNTROWS(fact_sales)
Total Quantity   = SUM(fact_sales[quantity])
Avg Ticket       = DIVIDE([Total Sales], [Total Orders])
Unique Customers = DISTINCTCOUNT(fact_sales[customer_id])
Max Order Value  = MAX(fact_sales[amount])
Min Order Value  = MIN(fact_sales[amount])
```

## Percentage of Total

A common pattern: what proportion does the current filter context represent of the whole.

```dax
-- Sales % of all products (removes product filter, keeps everything else)
Sales % of Total Products =
DIVIDE(
    [Total Sales],
    CALCULATE([Total Sales], REMOVEFILTERS(dim_product))
)

-- Sales % of selected period vs all time
Sales % of All Time =
DIVIDE(
    [Total Sales],
    CALCULATE([Total Sales], REMOVEFILTERS(dim_date))
)
```

## Conditional Aggregations

Aggregate only rows that meet a condition — without using `FILTER` on the full table.

```dax
-- Sales for a specific category
Sales Electronics =
CALCULATE([Total Sales], dim_product[category] = "Electronics")

-- Count of orders above a threshold
Large Orders =
CALCULATE(
    COUNTROWS(fact_sales),
    fact_sales[amount] >= 1000
)

-- Sales excluding returns
Net Sales =
CALCULATE(
    [Total Sales],
    fact_sales[order_type] <> "Return"
)
```

## Running Total

Accumulates a measure from the beginning of the visible range to the current row — useful for cumulative charts.

```dax
Sales Running Total =
CALCULATE(
    [Total Sales],
    FILTER(
        ALLSELECTED(dim_date[date]),
        dim_date[date] <= MAX(dim_date[date])
    )
)
```

`ALLSELECTED` respects the user's date slicer selection — the running total resets to the visible range rather than going back to the beginning of all time.

## Dynamic Titles

Titles that respond to slicer selections — useful for making reports feel responsive and self-describing.

```dax
-- Single value selected or default
Report Title =
"Sales Report — " & SELECTEDVALUE(dim_date[year], "All Years")

-- Multiple values selected
Selected Categories =
VAR selected = SELECTEDVALUE(dim_product[category], "All Categories")
RETURN
    "Category: " & selected

-- Count of selected items
Filter Summary =
VAR selected_count = COUNTROWS(ALLSELECTED(dim_product[product_name]))
VAR total_count    = COUNTROWS(ALL(dim_product[product_name]))
RETURN
    IF(
        selected_count = total_count,
        "All Products",
        selected_count & " of " & total_count & " products selected"
    )
```

## Dynamic Measures with Field Parameters

Field parameters allow report users to switch the measure displayed in a visual using a slicer — without the developer building separate visuals for each metric.

### Setting up a field parameter

Field parameters are created in Power BI Desktop via **Modeling → New Parameter → Fields**. Power BI generates a calculated table automatically:

```dax
-- Auto-generated by Power BI when creating a field parameter
Metric Parameter =
{
    ("Total Sales",   NAMEOF('Measures'[Total Sales]),   0),
    ("Total Orders",  NAMEOF('Measures'[Total Orders]),  1),
    ("Avg Ticket",    NAMEOF('Measures'[Avg Ticket]),    2),
    ("Unique Customers", NAMEOF('Measures'[Unique Customers]), 3)
}
```

### Using the parameter in a measure

```dax
-- Reference the selected field parameter value in a title
Selected Metric Title =
"Showing: " & SELECTEDVALUE('Metric Parameter'[Metric Parameter])
```

Add the parameter field to the visual's Values well, and add the parameter's name column to a slicer — the slicer now controls which measure the visual displays.

!!! note
    Field parameters were introduced in May 2022 and are now generally available. They replace many patterns that previously required complex `SWITCH` + `SELECTEDVALUE` workarounds.

## SWITCH-Based Dynamic Measures

Before field parameters, and still useful for more complex logic, `SWITCH` with `SELECTEDVALUE` was the standard pattern for switching between measures based on a slicer.

```dax
-- Disconnected slicer table drives which measure is shown
Selected Metric =
SWITCH(
    SELECTEDVALUE('Metric Selector'[Metric Key]),
    "sales",   [Total Sales],
    "orders",  [Total Orders],
    "ticket",  [Avg Ticket],
    [Total Sales]  -- default if nothing selected
)
```

This requires a disconnected table (not joined to any fact table) with one row per metric option. The slicer is built from this table.

!!! tip
    Use field parameters for simple metric switching — they are easier to set up and maintain. Use `SWITCH` + `SELECTEDVALUE` when the switching logic involves conditional formatting, custom labeling, or more than just selecting a measure.

## Conditional Formatting Values

DAX measures can drive conditional formatting in visuals — background color, font color, and data bars — making it easy to highlight exceptions without custom visuals.

```dax
-- Background color for a KPI card: green if above target, red if below
KPI Color =
IF(
    [Total Sales] >= [Sales Target],
    "#2d8a57",   -- green
    "#c0392b"    -- red
)

-- Three-tier color scale
Performance Color =
VAR variance_pct = DIVIDE([Total Sales] - [Sales Target], [Sales Target])
RETURN
    IF(variance_pct >= 0.05,  "#2d8a57",   -- above target
    IF(variance_pct >= 0,     "#f39c12",   -- on target
                              "#c0392b"))  -- below target
```

Apply these in Power BI Desktop via **Format visual → Cell elements → Background color → fx → Field value**.

## Top N + Others

Show the top N items in full and group everything else into an "Others" bucket.

```dax
-- Rank each product
Product Rank =
RANKX(ALL(dim_product[product_name]), [Total Sales], , DESC, DENSE)

-- Show name for top N, "Others" for the rest
Product Label =
IF([Product Rank] <= 5, SELECTEDVALUE(dim_product[product_name]), "Others")

-- Aggregate all "Others" correctly
Sales Top 5 and Others =
IF(
    [Product Rank] <= 5,
    [Total Sales],
    CALCULATE([Total Sales], FILTER(ALL(dim_product), [Product Rank] > 5))
)
```

## Last Refresh Timestamp

Display when the dataset was last refreshed — a common request from business users.

```dax
-- Calculated table updated at every refresh
Last Refresh =
ROW("Last Refresh", NOW())
```

Then create a measure:

```dax
Last Refreshed =
"Last updated: " & FORMAT(MAX('Last Refresh'[Last Refresh]), "DD MMM YYYY HH:MM")
```

Place this in a card visual on every report page.

## Best Practices

- Create explicit measures for every aggregation — do not rely on implicit column summation
- Use `SELECTEDVALUE` with a default value for all dynamic title measures — handles no-selection and multi-selection gracefully
- Use field parameters for metric switching when the logic is straightforward — simpler to maintain than `SWITCH` patterns
- Keep conditional formatting measures simple — one measure per formatting rule, named clearly (`KPI Background Color`, `Variance Font Color`)
- Always test dynamic measures with no selection, one selection, and multiple selections in the slicer
- Build a Last Refresh measure on every model — business users always ask
